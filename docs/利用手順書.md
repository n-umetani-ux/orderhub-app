# 利用手順書: 注文書管理システム

## 1. ログイン

1. ブラウザで https://orderhub-app-kappa.vercel.app にアクセス
2. 「Google でサインイン」ボタンをクリック
3. 会社のGoogle Workspaceアカウント（@beat-tech.co.jp）でログイン
4. ダッシュボードが表示されます

※個人のGoogleアカウントではログインできません。

---

## 2. ダッシュボード（担当案件・ギャップ管理）

### 2.1 画面の見方

上部サマリーカード:

| カード | 意味 |
|--------|------|
| 当月ギャップ | 今月稼働中だが注文書がないエンジニア数 |
| 来月切れ | 今月は注文書あるが、来月分がないエンジニア数 |
| 正常カバー | 注文書が正常にある人数 |
| 監視中 | 監視対象の全エンジニア数 |

カレンダーテーブル:

各行がエンジニア、各列が月を表します。

| 表示 | 意味 |
|------|------|
| ○（緑） | その月の注文書あり |
| 未（赤） | その月の注文書なし（対応が必要） |
| —（グレー） | その月は稼働対象外 |

月ヘッダーの青いドット:
稼働データが読み込まれている月に青いドットと下線が付きます。
例: 4月・5月に青ドット = 4月と5月の稼働一覧を読み込み済み。

### 2.2 フィルタ機能

| フィルタ | 操作 |
|---------|------|
| 拠点 | プルダウンで「全拠点/東京/大阪/福岡」を選択 |
| 自分の担当のみ | チェックを入れると自分の担当案件だけ表示 |
| 検索 | 氏名・社員番号・顧客名で絞り込み |

### 2.3 データの更新

- データは5分ごとに自動更新されます
- タブを切り替えて戻った時も自動更新されます
- 手動で更新したい場合は右上の「データを同期」ボタンをクリック
- 「再認証が必要です」と表示されたらボタンをクリックして再ログイン

## 3. 注文書のアップロード

### 3.1 ダッシュボードから登録する場合

1. ダッシュボードで対象エンジニアの行にある「登録」ボタンをクリック
2. アップロード画面に移動し、対象者が自動選択されます

### 3.2 アップロード画面から直接登録する場合

1. 左サイドバーの「新着アップロード・更新」をクリック

### 3.3 PDFの登録手順

ステップ1: ファイルの登録

1. 「ファイルを選択（複数可）」ボタンをクリック、またはPDFをドラッグ&ドロップ
2. PDFからテキストが自動抽出され、契約期間が検出されます

ステップ2: 情報の確認・入力

| 項目 | 説明 |
|------|------|
| 対象区分 | 「社員番号」「BP番号」「multi(チーム)」から選択 |
| 対象者 | 社員番号または氏名で検索して選択（顧客コード・顧客名が自動入力されます） |
| 部署 | プルダウンから選択 |
| 契約開始日 | PDFから自動入力（修正可能） |
| 契約終了日 | PDFから自動入力（修正可能） |
| 発行日 | PDFから自動入力（修正可能） |

ステップ3: ファイル名の確認

- 画面下部にリネーム後のファイル名がリアルタイム表示されます
- 命名規約: YYMMDD_部署コード_顧客コード顧客略称_社員番号.pdf

ステップ4: アップロード

1. 「アップロード」ボタンをクリック
2. Google Driveへの保存と注文書台帳への記録が自動で行われます
3. 完了画面で「Driveで確認」「保存フォルダを開く」「一覧へ戻る」から選択

### 3.4 multi（チーム）の場合

1. 対象区分を「multi(チーム)」に切り替え
2. 顧客コードをプルダウンから選択
3. 対象メンバーが自動検出されます（PDFに名前がある場合）
4. 連番を選択:
   - 0 = _multi.pdf（1枚目）
   - 1 = _multi-1.pdf（2枚目）
   - 2 = _multi-2.pdf（3枚目）...

---

## 4. Google Driveフォルダの確認

- サイドバーの「注文書PDF フォルダ」をクリック → Google Driveが開きます
- アップロード完了画面の「保存フォルダを開く」でも確認できます

---

## 5. 管理者向け機能

※管理者のみ利用可能な機能です。一般ユーザーには表示されません。

### 5.1 管理者設定モーダル

1. サイドバーの「管理者設定」をクリック
2. モーダルが開きます

管理者メールアドレスの編集:
- テキストエリアにカンマ区切りでメールアドレスを入力
- 例: n-umetani@beat-tech.co.jp,a-kudo@beat-tech.co.jp
- 「保存」をクリック

注意: 自分のメールアドレスを削除すると管理者権限を失います。
復旧するにはスプレッドシートの「設定」シートを直接編集してください。

PDF保存先フォルダの変更:
- Google DriveのフォルダIDまたはURLを入力
- 「保存」をクリック

### 5.2 手動オーバーライド

ダッシュボードのカレンダーセルを右クリックするとメニューが表示されます。

| 選択肢 | 効果 |
|--------|------|
| ○ カバー済み | 注文書ありとして強制表示 |
| 未 ギャップ | 注文書なしとして強制表示 |
| — 対象外 | 稼働対象外として強制表示 |
| 自動（解除） | オーバーライドを解除し、自動判定に戻す |

- オーバーライドされたセルには右上に紫色の小さなドットが付きます
- オーバーライドはスプレッドシートに保存されるため、全ユーザーに反映されます

### 5.3 管理者の追加・削除

管理者設定モーダルの「管理者メールアドレス」欄で追加・削除できます。
スプレッドシートの「設定」シート（adminEmails の行）を直接編集しても同じです。

---

## 6. よくある操作

| やりたいこと | 操作 |
|-------------|------|
| 自分の担当だけ見たい | 「自分の担当のみ」チェックを入れる |
| 特定のエンジニアを探したい | 検索欄に氏名・番号・顧客名を入力 |
| 注文書を登録したい | 「未」のエンジニアの「登録」ボタンをクリック |
| PDFの保存先を確認したい | サイドバーの「注文書PDF フォルダ」をクリック |
| データを最新にしたい | 右上の「データを同期」をクリック |
| ログアウトしたい | サイドバー下部の「ログアウト」をクリック |

---

## 7. トラブルシューティング

| 症状 | 対処法 |
|------|--------|
| 「再認証が必要です」と表示 | ボタンをクリックして再ログイン |
| データが表示されない | 「データを同期」ボタンをクリック |
| PDFからテキストが抽出できない | 手入力フォームに切り替わるので、契約期間等を手で入力 |
| 管理者設定が表示されない | 設定シートに自分のメールが adminEmails に登録されているか確認 |
| アップロードに失敗する | Driveの容量・フォルダのアクセス権限を確認 |
